## What's new in v1.0.8

### Direct MIDI output — make Matilda drive any synth in FL Studio (and any DAW)

FL Studio 20.0.1 does not forward MIDI **output** from VST3 plugins to other plugins, which is why Matilda was silent even though her engine fired notes and the playhead moved. This is the exact same path the **Standalone app** uses to drive GarageBand: it sends notes to a **virtual MIDI port**.

Matilda now does this from the plugin too:

- New **"MIDI Out"** selector in the bottom-left of the plugin window.
- Pick a virtual MIDI port (**loopMIDI** on Windows, **IAC Driver** on macOS) and Matilda streams her generated notes to it — *in addition* to the host MIDI buffer.
- Point your synth/instrument channel's **MIDI input** at the same port and it plays.
- The selected port is **saved with the project** and restored on reload.
- Newly created ports appear automatically (no need to reload the plugin).

This works on **every DAW and every FL Studio version** because it bypasses the host's VST3 MIDI routing entirely.

### How to use it in FL Studio

1. **Windows:** install [loopMIDI](https://www.tobias-erichsen.de/software/loopmidi.html) and create one port (e.g. `Matilda Out`).
   **macOS:** open *Audio MIDI Setup → MIDI Studio → IAC Driver → enable "Device is online"* and add a port.
2. Load Matilda, open its window, and set **MIDI Out** to that port.
3. Add your synth (e.g. on a separate channel). In the synth/channel **MIDI Input** settings, select the same port.
4. Press play — Matilda's notes now reach the synth.

> Note: host-side MIDI routing still works on DAWs that support VST3 MIDI out (e.g. newer FL, Bitwig, Reaper), so nothing changes there. The MIDI Out port is the reliable fallback for FL 20.x.

## Install

### macOS
- **VST3:** unzip `Matilda-macOS-vst3.zip` → copy `Matilda.vst3` to `~/Library/Audio/Plug-Ins/VST3/`
- **Standalone:** unzip `Matilda-macOS-standalone.zip` → open `Matilda.app`

### Windows (FL Studio, etc.)
- **VST3:** unzip `Matilda-Windows-vst3.zip` → copy `Matilda.vst3` to `C:\Program Files\Common Files\VST3\`
- **Standalone:** unzip `Matilda-Windows-standalone.zip` → run `Matilda.exe`
- Rescan plugins after copying the VST3 folder.
