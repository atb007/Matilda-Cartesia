## What's new in v1.0.9

This release is focused on the **macOS / Windows Standalone app**, which now lives in its own codebase (`matilda/standalone`) built on the stable v1.0.2 audio engine. The VST3/AU plugin is unchanged from v1.0.8.

### Standalone: tempo now syncs to your DAW over MIDI clock

Previously the standalone stayed stuck on the 120 BPM fallback even when MIDI clock was arriving from the DAW. The clock interval was measured in whole audio-block chunks, which produced out-of-range tempo estimates that were silently discarded.

- **Sample-accurate clock tempo** — inter-clock intervals are now timed using each clock message's exact in-buffer position, so the BPM tracks the DAW correctly (FL Studio "Send master sync", Ableton, etc.).
- **Reliable manual BPM** — when **"Sync external transport" is OFF**, incoming clock is ignored and the editable BPM field in the footer applies immediately. (Click it and type a value.)
- **No more freezing** — if clock stalls for more than half a second while synced, the sequencer falls back to its internal clock at the last known tempo instead of stopping dead.
- The footer BPM field now sits above the resize grips, so it's actually clickable.

### Standalone: forward-ported UI fixes from the plugin (no DAW-sync feature)

- **Filigree rendering** — Movement, Quantise Scale and Transport panels now use native SVG filigree with correct top/bottom flip and mirrored side ornaments (the v1.0.7 fixes).
- **Window state persistence** — the standalone remembers its UI scale (resize) and collapsed/expanded state across sessions.

### Two-port wiring recap (FL Studio + loopMIDI / IAC)

- **Port A (clock, DAW → Matilda):** DAW sends master sync to a virtual port; enable it as Matilda's MIDI input and turn on **Sync external transport**.
- **Port B (notes, Matilda → DAW → synth):** set Matilda's MIDI **output** to a second virtual port; in the DAW, point your synth channel's MIDI **input** port at the same port number.

## Install

### macOS
- **Standalone:** unzip `Matilda-macOS-standalone.zip` → open `Matilda.app`
- **VST3:** unzip `Matilda-macOS-vst3.zip` → copy `Matilda.vst3` to `~/Library/Audio/Plug-Ins/VST3/`

### Windows (FL Studio, etc.)
- **Standalone:** unzip `Matilda-Windows-standalone.zip` → run `Matilda.exe`
- **VST3:** unzip `Matilda-Windows-vst3.zip` → copy `Matilda.vst3` to `C:\Program Files\Common Files\VST3\`
- Rescan plugins after copying the VST3 folder.
